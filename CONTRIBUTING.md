# 贡献指南

欢迎参与！最受欢迎的几类贡献：

## 1. 适配其他输出后端

目前写入飞书多维表格。欢迎 PR：Notion、Airtable、本地 SQLite、Markdown 汇总等。
建议按 `ideainbox/feishu.py` 的模式新增独立模块，让 `push` 可切换后端。

## 2. 提炼模板

不同创作者风格不同。欢迎在 docs 里补充不同领域/风格的提炼提示词模板。

## 3. 输入源扩展

目前从本地文件夹读。欢迎适配：飞书文档、Notion 页面、邮件等作为投递入口。

## 开发约定

- **纯标准库优先**：核心逻辑不引第三方依赖。
- **不硬编码密钥**：用户私有信息走 `config/config.json`（gitignore）。
- **隐私优先**：`inbox/` 内容不进 git。
- **不编造**：提炼环节严禁编造证据/数据/来源。
- 提交前跑 `python -m ideainbox scan` 确保不报错。

## 提交流程

1. Fork & 新建分支
2. 改动 + 本地验证
3. PR 说明动机与验证方式

感谢 🙏
